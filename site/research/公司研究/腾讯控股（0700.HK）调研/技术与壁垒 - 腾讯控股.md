> **比较边界**
> 先动态建立这个行业的技术评价框架，再选择指标和标杆。严格同口径比较与代理比较必须分开；口径不一致时不强行排名，但仍要保留可确认的方向性事实、差距、未知项和核验路径。

技术研究的主线是：`技术能力 -> 相对水位 -> 客户可感知差异 -> 采购选择 -> 复制与切换阻力 -> 利润捕获 -> 财务验证 -> 可持续竞争优势`。技术领先不一定形成竞争优势；技术水位只是比较证据，技术壁垒解释优势为何能够持续，客户选择和商业结果才是最终验证。

## 11. 技术如何形成竞争优势与壁垒

竞争优势回答“客户为什么选它、公司为何能赢”，技术壁垒回答“竞对为什么难以复制或替代”。技术水位高不自动等于壁垒，更不自动等于竞争优势。壁垒结论只使用 `已验证壁垒 / 正在形成 / 仅为假设 / 不是壁垒`；竞争优势结论只使用 `已验证竞争优势 / 正在形成竞争优势 / 仅为竞争优势假设 / 不构成竞争优势`。

每项假设依次检查客户任务与付费方、相对竞对差异、水位证据、采购影响、客户采用/复购、复制资源与时间、切换成本、利润捕获、商业/财务验证、持续条件和反证。技术仅接近标杆时，若成本、兼容性、交付、服务或系统集成明确改变客户采购并得到商业验证，仍可能形成竞争优势；参数领先但客户不采用时不得认定。

证据极性使用固定结论：相对竞对差异只填 `已验证客户可感知优势 / 正在验证 / 未验证 / 无客户可感知优势`；采购影响只填 `已验证改变采购 / 正在验证 / 未验证 / 未改变采购`；客户采用只填 `已验证采用/复购 / 正在验证 / 未验证 / 未采用/未复购`；利润捕获只填 `已验证利润捕获 / 正在验证 / 未验证 / 未捕获利润`；财务验证只填 `已验证财务兑现 / 正在验证 / 未验证 / 未财务兑现`。明确反证必须进入对应结论和反证字段，不能因为证据栏有文字就当作支持。
