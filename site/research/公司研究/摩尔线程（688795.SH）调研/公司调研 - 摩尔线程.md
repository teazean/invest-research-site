## 1. 一句话看懂公司

摩尔线程是一家采用 Fabless 模式的全功能 GPU 公司。它自己设计芯片、MUSA 架构和软件栈，把晶圆制造、封装测试及部分板卡加工交给供应商，再向智算中心、云厂商、运营商、科研和政企客户销售板卡、一体机、服务器及集群。所谓“全功能”，是同一套 GPU 架构不只做 AI 训练和推理，还覆盖图形渲染、科学计算、视频编解码和虚拟化；这给公司更大的市场选项，也意味着研发战线比只做 AI 加速的公司更宽。

公司整体已经跨过“只有概念和样片”的阶段。2025 年收入 15.06 亿元，同比增长 243.37%，其中云端产品组合收入 14.61 亿元，占 97.04%；公司另行披露 S5000 已形成大规模商业落地、夸娥万卡集群已投入使用，但没有披露各型号或集群项目分别贡献多少收入、回款和复购，因此两类证据不能直接合并归因。2026Q1 收入进一步达到 7.38 亿元，同比增长 155.35%。但商业稳定性仍不足：2025 年四个季度收入分别为 2.89、4.13、0.83、7.21 亿元，Q3 毛利率接近零，Q4 又大幅恢复。报表只能确认季度收入和毛利剧烈波动；公司未单独解释 Q3 原因，项目交付、验收、产品组合、售价、成本结转或减值都可能影响结果，不能锁定某一项。

**2026Q1 的“归母净利润转正”不能理解为主营业务已经盈利。** 当季归母净利润 0.29 亿元，但扣非归母净利润仍为 -0.54 亿元；政府补助等非经常性收益合计约 0.84 亿元，超过账面利润。同时经营现金流为 -14.87 亿元，存货、预付款分别升至 21.95、23.61 亿元。底层原因是公司在为未来生产和交付先采购、预付，现金先流向供应链；只有产品卖出、验收并回款，才完成经营闭环。

管理层已经展示把研发推进到多代产品和项目的能力，但公司上市后的公开观察期还不足一个完整经营周期，因此总评不是“一般偏可靠”，而是“**观察期不足，无法充分判断；低至中置信度**”。创始人履历、多代产品和汇总收入是正面证据；但 S5000/万卡项目各自收入、回款和复购未披露，经营现金仍大额流出，“产品推进”尚未转成“稳定商业兑现”。关联采购也需纠正口径：年报附注列示 S2、W、Y 合计约 5.70 亿元；第 38 页的 4.78 亿元/21.63% 只是前五供应商中的关联采购（S2 子集）。按该页反推采购总额，全部关联采购占比约 25.79%，该比例是本文计算值，不是公司直接披露。关联预付合计约 13.42 亿元，金额足以要求持续审视履约、定价和资金安全。

股票价格比公司经营更激进。按 648.17 元和 4.70028217 亿股估算，总市值约 3,046.6 亿元，对应 2025 年 PS 约 202.4 倍、TTM PS 约 155.9 倍、2026Q1 期末净资产 PB 约 26.4 倍；静态和TTM都亏损，可核验外部预测样本对2026年也仍为亏损，正常化利润尚未形成，所以四种PE目前均不能给出有经济意义的正数。要在2028年以75倍PE只支撑当前名义市值，公司需实现约40.6亿元净利润；若净利率20%，对应约203.1亿元收入，是2025年收入的13.5倍。若还要求年化10%回报，净利润和收入门槛分别升至约51.4亿元和257.0亿元。这不是“不可能”，但明显属于多项乐观假设同时成立的路径。

**综合判断：摩尔线程是值得持续深究的国产 GPU 平台公司，但当前股票属于“公司进步很快，价格要求更快”。研究重点应放在收入能否跨季度平滑、扣非盈利和现金流何时同步转正、关联预付款能否转成可销售库存、客户集中度能否下降，以及 2026 年 9 月和 12 月解禁后估值是否仍有承接。**

## 2. 生意模式：AI 需求怎样变成摩尔线程的利润

| 环节 | 发生了什么 | 原因和机制 | 谁付钱 | 谁受益 | 投资含义 | 反证 |
|---|---|---|---|---|---|---|
| AI 使用量增长 | 训练、推理和仿真可能需要更多计算 | 模型变大、用户和调用次数增加会提高芯片、显存、网络和机房需求；实际增量还要扣除算法效率提升 | 云厂商、互联网平台、运营商、科研及政企客户 | GPU、服务器、网络和数据中心供应链 | 需求机制成立，行业实际增长幅度仍需资本开支、使用量和订单验证 | 算法效率提升快于调用量，或客户削减资本开支 |
| 客户选 GPU | 客户不只看峰值算力 | 真正成本还包括模型迁移、算子覆盖、集群利用率、稳定性、能耗和运维 | 算力采购方 | 能交付完整软硬件系统的厂商 | MUSA 软件和集群工程与芯片参数同样重要 | 迁移慢、故障多、有效算力低或总体成本不具优势 |
| 摩尔线程交付 | 销售板卡、一体机和集群 | 直销便于做大项目，经销扩大覆盖；项目验收决定确认收入的时点 | 大型机构客户及经销商 | 公司和上游代工/设备供应商 | 大单能让收入迅速跳升，也造成季度波动和客户集中 | 客户延迟验收、取消项目或转向竞对 |
| 毛利覆盖研发 | 2025 毛利率 65.57%，但仍亏损 | GPU 研发和软件生态前置投入大；收入规模还不足以覆盖研发、管理和股份支付 | 最终由客户采购款承担 | 规模扩大后公司、员工和股东受益 | 收入增长快于费用时会产生经营杠杆 | 毛利下降、研发战线过宽或价格竞争加剧 |
| 利润转成现金 | 客户回款并消化库存/预付 | 采购和预付通常早于销售；会计收入也可能早于现金回收 | 客户付款 | 公司现金流 | 现金流比“某季转盈”更接近商业闭环 | 存货、预付持续快于收入增长，减值或坏账上升 |

### 常用术语

| 术语 | 小白解释 |
|---|---|
| Fabless | 自己设计芯片，不自建晶圆厂；制造和封测交给供应商 |
| GPU | 擅长并行计算的处理器，既可做图形，也可做 AI 和科学计算 |
| MUSA | 摩尔线程自研的统一芯片架构、编程模型、软件库和驱动体系 |
| CUDA | NVIDIA 的 GPU 编程平台和生态；开发者、工具和代码积累是核心壁垒 |
| 算子 | 模型里的基础计算单元；算子不全或效率低，理论算力就用不出来 |
| MFU | 模型计算利用率，衡量训练时理论算力有多少真正用于模型计算 |
| 线性扩展效率 | 加卡后性能能否接近同比例提高；规模越大越难 |
| AllReduce | 多张卡把各自的计算结果汇总后再分发的集合通信；是集群是否堵车的关键检查 |
| 墙钟时间（wall-clock time） | 从任务开始到达到指定质量的真实经过时间；比理论峰值更接近客户实际等待 |
| TTFT / TPOT | TTFT 是推理时第一个 token 的等待，TPOT 是后续每个 token 平均等待；一个影响首次响应，一个影响连续输出 |
| 扣非净利润 | 去掉政府补助、金融资产收益等非经常项目，更接近主营经营结果 |
| PS / PB | 市值分别除以收入、净资产；亏损公司 PE 失效时常用，但不能证明便宜 |
| Dense / MoE | Dense 每次计算通常激活全部主要参数；MoE 只激活部分“专家”，两者对通信和算力利用要求不同 |
| PH100 | S5000 所搭载的摩尔线程芯片型号，不是另一家独立产品线 |
| SoC | 把 CPU、GPU、内存接口等多个模块集成到一颗芯片的系统级芯片 |
| AIBOOK | 摩尔线程基于“长江”SoC 推出的端侧 AI 计算设备 |
| TCO | 总拥有成本：硬件价格加迁移、能耗、机房、运维和故障成本 |
| TTM / CAGR | TTM 是最近十二个月；CAGR 是一段时间内折算后的年均复合增速 |
| FCF | 自由现金流，本文简化口径为经营现金流减长期资产购置支出 |
| ROCm / NVLink | ROCm 是 AMD 的 GPU 软件平台；NVLink 是 NVIDIA 的高速芯片互联技术 |
| 2P4D / prefill / decode | 2P4D 是公司某次推理测试采用的“两组输入处理、四组逐字生成”资源配置；prefill 是先读完输入，decode 是随后逐个生成输出，两阶段瓶颈不同 |
| KV Cache | 大模型推理时保存前文计算结果的缓存，能减少重复计算，但会占用大量显存 |
| HBM3E | 新一代高带宽显存；带宽和容量会影响大模型能否装下及喂给芯片数据的速度，但不能单独代表整机性能 |
| EDA / IP | EDA 是芯片设计软件，IP 是可复用的电路模块；受限会影响设计、验证和流片 |
| MCCL | 摩尔线程多卡通信库，作用类似“让多张卡交换数据的交通规则”；存在不等于在所有规模下都稳定高效 |
| SLA / Day-0 | SLA 是服务响应和可用性承诺；Day-0 表示新模型发布当天即可支持，必须用版本记录和实际运行验证 |
| MTTR | 平均修复时间；集群出故障后多久恢复，数值越低通常越好 |
| CTS / SPECviewperf | CTS 用来检查图形 API 实现是否符合规范；SPECviewperf 用专业图形工作负载测性能。“支持 Vulkan”不等于两项都达标 |
| ASP | 平均售价；收入大致由销量/项目数乘 ASP 形成，不拆开就不知道增长来自多卖还是涨价 |
| “匿名国际旗舰 74%+” | 公司自测对照未给出具体型号和完整任务口径；只是能力线索，不能被写成“达到全球旗舰 74%”的已验证行业水位 |
| BOM / HPC | BOM 是一套产品全部物料成本；HPC 是高性能计算，如科学仿真，不等同于大模型训练 |
| MaaS | 模型即服务，客户按接口或用量购买模型能力，不必自己购买和维护芯片集群 |
| FP4 / FP8 / FP64 | 数字精度等级；位数较低通常更省显存、算得更快，但可能损失准确性，必须用真实模型验证 |
| TOPS / TFLOPS | 芯片每秒理论运算次数的单位；只能描述特定精度的理论上限，不能替代真实模型速度 |
| batch / SDK | batch 是一次并行处理的请求数量；SDK 是开发工具包，帮助程序调用芯片能力 |
| CANN / PAI | CANN 是华为昇腾的软件平台；PAI 是阿里云机器学习平台，二者都是竞对全栈的一部分 |
| SXM / NPU / VPU | SXM 是 NVIDIA 的高功耗高互联模组形态；NPU 偏 AI 计算，VPU 偏视频处理 |
| AI4S | 用人工智能辅助科学研究，例如材料、药物和物理仿真 |
| 协定存款 | 银行账户按约定条件计息、通常可支取的存款安排；收益和流动性要看具体协议 |

## 5. 核心能力、竞争优势与技术壁垒

本文不从公司恰好披露的参数反推“行业看什么”，而是先问付费客户要完成什么任务，再确定每个技术方向的指标、单位、测量口径、好坏方向和国内/全球标尺。对摩尔线程，AI 训练看达到目标质量的墙钟时间，推理看质量约束下的吞吐、TTFT 和 TPOT，计算内核先校准精度/稀疏/引擎和正确性，内存与互联区分峰值和有效工作负载。软件要看通过率、时滞、迁移人天和故障；“全功能 GPU”还必须分别回答图形与驱动、HPC、视频编解码、vGPU/云桌面、安全认证和客户验收。能效、TCO、可靠性和供货最终把技术变成客户是否下单的经济问题。

成熟度分成技术可行、产品、工程和商业四层，只回答“做到哪一步”；相对水位单独回答“与谁、在哪个任务上相比如何”。两者都不能自动变成经济壁垒；壁垒还要有客户采用/复购、难复制资源、切换成本和利润捕获。公开资料无法量化复制时间时，就写无法判断，不凭感觉填“几年”。

| 技术/产品层 | 四层成熟度摘要 | 严格/代理水位 | why 与证据边界 | 仍需验证 |

产品代际还必须先分形态：花港是 GPU 架构，PH100/华山/庐山是芯片，S3000/S4000/S5000 是板卡，KUAE/C256 是集群或超节点，MUSA 是软件平台。只在同形态内说 N-1/N/N+1；架构发布不能跨层写成芯片量产，板卡交付也不能直接证明集群长周期稳定。

为什么“万卡”仍不能直接等于成熟盈利？公司披露的集群建成只是工程实践线索，缺少客户长周期运行和故障数据时，不能独立确认工程质量。投资回报还取决于客户是否持续付费、有效运行时间、维护成本和后续复购。技术落地是收入的前提，收入增长是覆盖研发的前提，回款才是现金闭环。

**代理水位的已知边界：** S5000 公开了 80GB 名义容量、1.6TB/s 峰值带宽和 784GB/s AI Link；H100、DGX B200 和 MI350 平台也有官方规格。容量和带宽可并列作静态代理，但产品形态、代际、可用保留、缓存和工作负载不同，不用峰值百分比生成任务性能排名。S5000 784GB/s 与 H100 900GB/s 的方向、聚合定义和拓扑未校准，所以连静态差距也不计算。真正的行业水位必须回到同模型/任务、精度、质量、软件、系统规模、功耗和价格下的任务结果。

先把概念分开：技术能力是能否做出来，技术水位是同一客户任务下相对竞对怎样，技术壁垒是竞对复制的资源和时间，竞争优势是客户为什么因此选择摩尔线程，商业护城河还要求这种选择持续转成份额、定价权、利润和现金。投资价值则要再问当前约 3,046.6 亿元市值是否已经计入这些结果。

| 股权激励 | 计划总量为股本 0.1615%；草案拟授予 1,080 人，2026-05-19 实际首次授予调整为 1,042 人，约占 2025 年末 1,274 名员工的 81.79%；2026/2027 满额归属只考收入 25 亿元、两年累计 70 亿元 | 广覆盖、低稀释利于留人；但不考毛利、利润、现金流，且门槛低于招股书盈利所需收入假设 | 股东承担稀释和费用，员工受益 | 收入达标但仍高亏、高现金流出 |
| 薪酬和股份支付 | 张建中 2025 税前薪酬 720 万元；关键管理人员报酬 3,510 万元；股份支付费用 3.53 亿元 | 高技术人才薪酬有行业背景，但亏损期成本不低，需与经营兑现匹配 | 公司和股东承担成本，管理与员工受益 | 费用持续高而收入、利润和产品不兑现 |
| 审计、诉讼与监管 | 2025 财务和内控均获标准无保留意见；年报披露无内控重大缺陷、非经营性资金占用、违规担保、重大诉讼仲裁及违法处罚 | 年报覆盖范围内未见基础治理红旗，但上市历史很短，且不能替代对所有司法/监管数据库的穷尽检索 | 全体股东受益 | 审计意见变化、重大问询、诉讼、内控缺陷、违规占用或处罚 |
| 用工合规 | 年报披露历史上存在部分员工社保、公积金未足额缴纳或通过第三方人力资源机构缴纳；实控人承诺若产生补缴或处罚将承担损失 | 这是已披露的合规瑕疵，不等同于重大治理失控；实控人补偿承诺降低公司损失，但仍需看是否实际执行 | 若发生追缴，实控人按承诺承担；若承诺无法执行，公司和股东承压 | 发生大额追缴、处罚，或实控人未履行补偿承诺 |

激励目标尤其值得小白理解：2026 年满额归属收入目标为 25 亿元，2027 年两年累计 70 亿元；若 2026 恰好 25 亿元，则 2027 只需 45 亿元。招股书却假设 2027 年收入 59.83 亿元、毛利率 61%并含 3.07 亿元政府补助，才可能实现合并盈利。也就是说，员工可以在公司尚未实现招股书所述盈利条件时满额归属，激励更偏“做大收入”，不是“替股东创造现金利润”。

### 管理层执行记录：看结果，不看履历光环

| 时段 | 管理层推进事项 | 可核验结果 | 为什么只给这一级评价 | 反证/下一步验证 |
|---|---|---|---|---|
| 2020-2023 | 建团队并推进多代 MUSA 架构和消费、云端产品 | 年报和招股书可确认多代研发项目及产品存在 | 证明能组织复杂研发并把部分项目做成产品；未披露各代良率、销量和盈利 | 后续产品延期、旧产品快速淘汰或客户不复购则降级 |
| 2023 实体清单后 | 调整供应链并继续迭代 | 之后仍推出云端产品并形成收入 | 证明业务没有立即停摆；大额预付和关联采购说明供应问题仍在付出现金与治理成本 | 工具/IP进一步受限、预付无法交付、存货减值或代际延期 |
| 2024 | 扩大云端商业化并进行上市前融资 | 收入增至 4.38 亿元、毛利转正，但净亏 16.18 亿元、经营现金流 -19.55 亿元 | 产品组合开始被客户付费，尚未覆盖研发和现金消耗 | 2025-2026 能否把收入增长转成扣非利润与回款 |
| 2025 | 云端产品组合放量并完成 IPO | 收入 15.06 亿元；但季度波动大、前五客户占 91.36%、经营现金流 -29.56 亿元 | 规模收入兑现是正面证据；稳定复购、分散客户和现金闭环尚未兑现 | 客户集中下降、旧客户扩容、现金流改善；反之降级 |
| 2026Q1 | 继续交付并推进下一代研发 | 收入 7.38 亿元、归母 0.29 亿元；扣非 -0.54 亿元、经营现金流 -14.87 亿元，预付和存货继续增加 | 表面盈利不能覆盖现金和营运资本问题，因此不把单季转正当作商业执行成熟 | 扣非和经营现金流连续转正、预付到货、库存销售并回款后才升级 |

这张记录说明为什么结论不是“管理层优秀”或“不靠谱”二选一：产品推进有连续结果，商业质量和治理却仍有明确缺口，而且上市后的公开观察期不足一个完整周期。因此当前总判断是“观察期不足，无法充分判断”，而不是把历史不足误写成不可靠。管理层评价要随未来 2-3 年的履约、回款、关联预付、危机处理和资本配置变化，而不是永久标签。

## 10. 股价、市值与市场预期反推

| 问题 | 回答 |
|---|---|
| 当前买的是什么 | 不是“能否转盈”，而是 2027-2028 年高速扩张、形成长期国产 GPU 头部并取得高利润率 |
| 四种PE口径 | 静态PE不适用（2025亏损10.01亿元）；TTM PE不适用（亏损8.59亿元）；Forward PE不适用（可核验机构样本仍预测2026亏损）；正常化PE不适用（尚未形成可验证正常化利润） |
| 市值有多高 | 约 3,046.6 亿元；2025 PS 约 202.4 倍，TTM PS 约 155.9 倍；PS只能描述价格相对收入，不能证明便宜或昂贵到何种程度 |
| 预期有多苛刻 | 约 203.1 亿元收入、20%净利率和 75 倍 PE 只够让 2028 市值名义上接近当前；若要求年化 10%回报，所需收入约 257.0 亿元、净利润约 51.4 亿元 |
| 最大预期差研究假设 | 一种待验证假设是：市场可能高估单季归母转正的主营含金量和收入连续性，低估预付/库存现金占用及解禁供给；没有一致预期数据时不把它写成已证实的“市场误判” |
| 正面催化 | 扣非转正、经营现金流改善、下一代产品量产、多客户复购、集中度下降 |
| 交易结构风险 | 当前流通约 3,022.55 万股，占总股本约 6.43%；9 月约 2,577.45 万股锁定到期；按上市公告逐项汇总，12 个月锁定批次约 1.8591 亿股，占 39.55%，实际流通日以届时公告为准 |

估值压力测试使用可复算公式：`情景终值的折现结果 = 目标年份每股价值 / (1 + 要求年化回报率)^持有年数`。以年化10%作为数学示例，而非适合所有人的统一标准，从2026-07-15到2028年末约2.47年。由于客户数、项目数、SKU、销量、ASP和成本结构均未披露，四个情景只用于展示敏感性，任何折现结果都不是“可接受价格”、目标价或购买候选阈值。只有先补齐可复核业务量价桥，再按投资者自己的期限与回报门槛计算，估值条件才可能成立。

完整反推和情景见[股价预期](股价预期%20-%20摩尔线程.md)。

## 11. 全文判断链条自检

| 判断 | 原因 | 机制 | 付费方/受益方 | 投资含义 | 反证 |
|---|---|---|---|---|---|
| 公司已跨过产品和规模收入门槛 | 2025 年云端收入 14.61 亿元，2026Q1 收入 7.38 亿元 | 客户验收板卡、一体机或集群后确认收入 | 算力采购方付费；公司和上游供应链受益 | 公司值得进入长期深挖池 | 产品级收入无法持续、老客户不复购、季度收入再次断层 |
| 技术能力存在，但经济壁垒仅为假设 | 有 S5000、MUSA 和公司披露的万卡部署，缺少同口径性能、迁移工时、项目收入和客户复购 | 能力必须降低客户 TCO、形成切换成本并被公司以份额、价格或毛利捕获，才成为经济壁垒 | 客户因效率和风险下降受益；公司只有在复购和利润中才能受益 | 不能把公司自测、汇总收入或专利数直接资本化为高估值 | 独立测试落后、迁移成本高、集群利用率低或不复购；相反证据出现后才上调 |
| 主营尚未完成盈利和现金闭环 | 2026Q1 扣非仍亏，经营现金流 -14.87 亿元，预付和存货继续上升 | 采购和预付先流出现金，只有交付、验收和回款后才闭环 | 供应商先收到资金；公司和股东承担周转与减值风险 | 归母单季转正不能等同成熟盈利 | 扣非和经营现金流连续转正，预付/库存周转改善且减值受控 |
| 管理层观察期不足，无法充分判断 | 多代产品推进与汇总收入是正面证据；具体产品复购、现金闭环仍未验证，关联采购/预付提高验证门槛；低至中置信度 | 权力集中可提高决策速度，也使定价、履约和资金安全更依赖治理 | 关联供应商先获得现金；中小股东承担履约和定价风险 | 不能只凭创始人履历或收入增长无条件信任 | 关联交易独立比价/履约、客户复购、利润与现金同步可证则上调；减值、问询或占用则下调 |
| 当前股价反映极乐观预期 | 约 3,046.6 亿元市值、约 202.4 倍 2025 PS，TTM 仍亏损 | 只有收入、净利率、头部地位和高估值长期同时成立，当前价格才有足够回报 | 股东承担预期落空和估值压缩；经营兑现者受益 | 好公司不自动等于当前价格的好股票 | 盈利远超反推路径或股价大幅下降并形成可验证安全边际 |

## 12. 公司关注价值与股票购买候选资格

| 判断对象 | 当前结论 | 底层原因 | 投资含义 | 反证/升级条件 |
|---|---|---|---|---|
| 公司关注价值 | 值得深挖 | 云端业务已形成收入，S5000 和集群有落地迹象；具体收入归因和重复商业化仍待验证 | 可进入长期研究池 | 客户复购、独立性能、产品收入拆分、扣非和现金流连续改善 |
| 管理层 | 观察期不足，无法充分判断；低至中置信度 | 产品推进已有连续结果；具体产品复购和现金闭环未验证，关联预付、收入型激励和上市历史短需折价 | 不能只凭履历或产品发布无条件信任 | 关联交易独立比价/履约、客户复购、利润与现金同步兑现才上调 |
| 股票购买候选资格 | 暂不具备 | 当前市值要求极高的收入、利润和估值同时成立，技术水位和现金闭环尚未完全验证 | 这是研究资格，不是当前交易动作；不能用“国产替代空间大”直接证明便宜 | 经营证据和估值条件同时满足后才升级 |
| 最关键变量 | 现金闭环 | 采购预付先于收入，经营现金流长期大负 | 决定 IPO 资金是增长燃料还是持续消耗 | 存货和预付周转、经营现金流、扣非利润 |

“进入购买候选”需要同时满足两组条件：经营上出现多客户复购、同口径技术证据、扣非与现金改善、预付和存货周转可控，并能用客户数、项目数、SKU、销量、ASP、毛利和现金转化搭出可审计业务桥；估值上再用折现公式判断预期回报是否达到投资者自己的门槛。当前第一组条件尚未完成，因此不能把任一作者情景的每股数学结果当作购买价格锚。若用户需要判断当前买入、持有、加仓、降低暴露或退出，必须把本报告的资格结论、价格和持有期限交给 `investment-decision-discipline`；本报告不自创交易动作。
