# AI产业链术语表

> 更新日期：2026-07-15。

这份术语表的目标不是写得学术，而是帮投资阅读时不被概念绕晕。每个术语都尽量用“小白话 + 投资含义”的方式解释。

| 术语 | 小白话解释 | 投资上为什么重要 |
|---|---|---|
| AI factory | 把数据、模型、算力和软件流程组合起来，持续产出 AI 服务的“智能工厂” | 说明 AI 不是单个模型，而是需要长期资本开支和运营能力 |
| GPU | 原本常用于图形计算，现在也非常适合做 AI 的并行计算芯片 | 当前 AI 算力核心硬件，利润池集中 |
| ASIC | 为特定任务定制的芯片 | 大客户可能用 ASIC 降低成本，分流部分 GPU 需求 |
| NPU | 常见于手机、PC、车等终端里的 AI 加速单元 | 端侧 AI 能不能普及，取决于 NPU 性能和软件生态 |
| TOPS | 每秒万亿次操作，常用来描述 NPU 或边缘 AI 芯片算力 | 能粗略比较端侧 AI 芯片能力，但不能单独代表真实体验 |
| HBM | 高带宽内存，像给 AI 芯片配的高速数据通道 | AI芯片再强，也需要 HBM 喂数据；供给紧张时议价强 |
| 先进封装 | 把芯片、HBM 等高密度连接在一起的制造技术 | 先进封装产能不够时，高端 AI芯片交付会受限 |
| CoWoS | 台积电的一类先进封装技术 | 常被市场当作 AI芯片供应瓶颈的代表之一 |
| AI服务器 | 装有 GPU/ASIC 等加速卡的高性能服务器 | 服务器订单能直接体现算力资本开支强度 |
| Rack / 机柜 | 数据中心里放服务器的一格“货架” | AI机柜功率密度高，会带动供电和液冷升级 |
| InfiniBand | 常用于高性能计算和 AI训练的高速网络技术 | 影响大规模训练集群的效率 |
| Ethernet / 以太网 | 通用网络技术，AI集群也在快速使用高性能以太网方案 | 若以太网方案扩张，会影响网络设备和芯片格局 |
| RoCE | 在以太网上实现接近高性能网络能力的一类技术 | 是 AI数据中心网络路线的重要组成 |
| 光模块 | 把电信号和光信号转换，用于高速数据传输 | AI集群规模越大，对高速光互连需求越强 |
| Switch / 交换机 | 网络中的交通枢纽，让服务器之间互相传数据 | 大模型训练需要大量服务器协同，交换机变成核心设备 |
| PUE | 数据中心总耗电除以 IT设备耗电，越接近 1 越节能 | PUE 影响电费和 ESG，也影响数据中心盈利能力 |
| TDP | 芯片设计功耗指标 | 功耗越高，散热和供电压力越大 |
| 液冷 | 用液体带走服务器热量的散热方式 | 高功率 AI机柜可能需要液冷，带动热管理产业链 |
| CDU | 液冷系统里的冷量分配单元 | 是液冷系统关键部件之一 |
| UPS | 不间断电源，电力异常时保护服务器 | AI服务器价值高，停机会造成巨大损失，UPS 需求更重要 |
| 变压器 | 改变电压，把电力送到合适层级 | 数据中心扩建会拉动变压器和配电设备需求 |
| PPA | 长期购电协议 | 数据中心要锁定稳定电力和电价，PPA 会影响成本和可持续性 |
| Edge AI / 端侧AI | 在手机、PC、汽车、摄像头、工厂设备等终端附近运行 AI | 可以降低延迟、减少云端推理成本、保护隐私，并带动终端硬件升级 |
| Hyperscaler | 超大规模云厂商，比如 Microsoft、Amazon、Google、Meta 等 | 他们的资本开支决定 AI基础设施周期 |
| NeoCloud | 新型 AI云服务商，通常专注 GPU算力租赁 | 增长快，但重资产、客户集中和融资风险也高 |
| Training / 训练 | 用大量数据让模型学会能力 | 训练需要大规模算力，偏资本开支驱动 |
| Inference / 推理 | 用户真正使用模型时，模型生成答案 | 推理量决定 AI能否从建设期进入持续收入期 |
| Token | 模型处理文本的基本单位，可以粗略理解为字词碎片 | API收费、模型成本和使用量常按 token 计算 |
| Context window | 模型一次能“看见”的上下文长度 | 长上下文会增加计算和显存需求，也会影响应用体验 |
| Agent | 能分解任务、调用工具、连续执行动作的 AI系统 | 如果 Agent 真正进入工作流，应用层价值会大幅提升 |
| Agentic Work Unit / AWU | Salesforce 用来描述 Agent 完成一次实际工作单元的指标 | 比单纯 token 更接近业务结果，但不同公司口径不可直接比较 |
| MCP | Model Context Protocol，可理解为让模型和外部工具、数据源、系统连接的一种协议思路 | Agent 想进入工作流，必须安全连接工具和数据，MCP 类协议会影响生态入口 |
| RAG | 检索增强生成，让模型先查资料再回答 | 企业知识库和专业场景常用，能降低幻觉 |
| Vector database | 向量数据库，用于存储和检索语义相近的信息 | RAG、企业知识库和 AI搜索的重要底座 |
| MLOps | 管理模型开发、部署、监控和迭代的一套流程 | 企业大规模使用 AI 时，工具链需求会提升 |
| Fine-tuning / 微调 | 在基础模型上用特定数据继续训练，让它更适合某个任务 | 影响企业定制化需求和模型服务收入 |
| VLA 模型 | Vision-Language-Action，视觉、语言和动作结合的模型路线 | 具身智能和机器人要从“看懂”走向“会做”，需要这类能力 |
| 世界模型 | 让 AI 对物理世界变化形成内部预测的一类模型思路 | 如果成熟，机器人和自动驾驶的规划能力可能提升 |
| 仿真训练 | 在虚拟环境里训练和测试机器人或自动驾驶系统 | 可以降低真实世界试错成本，但仿真到现实仍可能有差距 |
| 执行器 | 机器人关节和运动系统里的动力部件 | 决定机器人力量、速度、寿命和成本，是人形机器人核心零部件 |
| 末端执行器 | 机器人用来抓取、夹持、吸附或操作物体的末端工具 | 决定机器人能不能真正完成工位任务 |
| ARR | 年化经常性收入 | 判断 AI软件或云服务是不是形成持续收入 |
| RPO | 剩余履约义务，代表已签合同未来还没确认的收入 | 云和 AI基础设施公司常用来展示未来收入能见度 |
| PE / P/E | 市盈率，股价或基金持仓价格相对盈利的倍数 | 用来粗略判断贵不贵，但高成长公司不能只看 PE |
| PB / P/B | 市净率，股价或基金持仓价格相对净资产的倍数 | 对硬件、半导体和高研发公司只是辅助指标 |
| Beta | 衡量资产相对市场波动的指标 | AI主题和半导体 ETF beta 高时，回撤可能比大盘更剧烈 |
| Capex | 资本开支，比如买 GPU、建数据中心 | AI基础设施周期最核心指标之一 |
| Free cash flow / 自由现金流 | 经营现金流减资本开支后剩下的钱 | 判断高增长是否真的能转化成现金 |
| Backlog | 已签但未交付或未确认收入的订单积压 | 订单能见度高是好事，但也要看能否交付和收款 |
| Utilization / 利用率 | 算力设备实际被使用的比例 | 算力云能否赚钱，很大程度取决于利用率 |
| Net Revenue Retention / NRR | 老客户本期收入相对上一期收入的留存和扩张比例 | AI 软件如果能让老客户加购，NRR 会更好；如果只是试用，NRR 不一定改善 |
| AI-first ARR | Adobe 等公司用来描述 AI优先产品带来的年化经常性收入 | 可以观察 AI商业化，但各公司定义不同，不能直接横向相加 |

## 英文全称速查

| 缩写 | 英文全称 | 小白话 |
|---|---|---|
| GPU | Graphics Processing Unit | 图形处理器，现在也是 AI 并行计算核心芯片 |
| ASIC | Application-Specific Integrated Circuit | 为特定任务定制的芯片 |
| NPU | Neural Processing Unit | 端侧或设备里的 AI 加速单元 |
| HBM | High Bandwidth Memory | 高带宽内存 |
| CPO | Co-Packaged Optics | 共封装光学，把光互连和芯片更紧密集成 |
| PUE | Power Usage Effectiveness | 数据中心能源使用效率 |
| UPS | Uninterruptible Power Supply | 不间断电源 |
| PPA | Power Purchase Agreement | 长期购电协议 |
| RAG | Retrieval-Augmented Generation | 检索增强生成 |
| MLOps | Machine Learning Operations | 模型开发、部署、监控和迭代流程 |
| ARR | Annual Recurring Revenue | 年化经常性收入 |
| RPO | Remaining Performance Obligations | 剩余履约义务 |
| NRR | Net Revenue Retention | 净收入留存率 |
| PE / P/E | Price-to-Earnings Ratio | 市盈率 |
| PB / P/B | Price-to-Book Ratio | 市净率 |
| VLA | Vision-Language-Action | 视觉、语言、动作结合的具身智能模型路线 |

## 阅读提示

如果只想快速判断 AI 产业链，先记住四组词就够了：

第一组是“GPU、HBM、先进封装”。这组决定高端 AI芯片能不能做出来。

第二组是“服务器、网络、光模块”。这组决定很多芯片能不能连成可用集群。

第三组是“数据中心、电力、液冷”。这组决定算力能不能真正上线。

第四组是“云、大模型、应用、Agent”。这组决定前面花出去的钱能不能赚回来。

第五组是“ARR、RPO、NRR、自由现金流”。这组决定 AI热度是否真的进入财务报表，尤其能帮助判断收入增长是不是有质量。

第六组是“PE、PB、Beta、回撤、持仓集中度”。这组用来判断相关基金/ETF是不是已经很贵、很拥挤，避免把长期产业趋势误读成短期无风险买点。
