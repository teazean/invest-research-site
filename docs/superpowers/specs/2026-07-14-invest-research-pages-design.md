# 投资研究 GitHub Pages 自动发布设计

## 目标

把 `invest` Obsidian vault 继续作为唯一研究正文源，在不改变现有写作习惯的前提下，将公司研究与产业专题自动发布为手机友好的 GitHub Pages 网站。

首版成功标准：

- 公开站点地址为 `https://teazean.github.io/invest-research-site/`。
- 已有全部公司研究与产业专题均可访问，包括正文、配图和允许公开的 CSV。
- 中际旭创页面采用已确认的 B 方案：先看速览，再进入正文、财务、估值、竞对等完整材料。
- Obsidian Git 把内容推送到私有备份仓库后，后续筛选、转换、构建和部署完全在 GitHub 云端完成。
- 任一校验或构建失败时不覆盖当前线上版本。

## 系统边界与数据流

系统分为三个清晰边界：

1. **写作与备份**：`/Users/zhang/Documents/obsidian_vaults/invest` 和私有仓库 `teazean/obsidian-vault-invest` 保存完整 vault。现有 Obsidian Git 自动备份机制保持不变。
2. **公开内容加工**：私有仓库的 `publish-research.yml` 在 `master` 推送且目标目录变化时触发，调用公开站点仓库提供的同步程序，只生成允许公开的内容副本。
3. **网站构建与托管**：公开仓库 `teazean/invest-research-site` 保存站点程序、公开内容副本和 Pages 工作流；GitHub Actions 构建静态 HTML/CSS/JavaScript，GitHub Pages 只负责托管构建产物。

数据流固定为：

```text
Codex/Obsidian 写 Markdown
  → Obsidian Git 推送私有备份仓库
  → 私有仓库 Action 筛选并同步公开内容
  → 公开仓库 Action 构建静态站点
  → GitHub Pages 发布
```

自动发布不调用大模型。本地只负责写作和首次推送；私有仓库收到推送后，本地可以离线。

## 内容公开规则

允许公开的根目录只有：

- `投资研究/公司研究/**`
- `投资研究/产业专题/**`

同步程序采用默认拒绝策略：

- 包含标准 GFM Markdown、`assets/` 图片和 `csv/` 结构化研究数据。
- 排除 `reports/` 财报 PDF、`data/` 原始抓取数据、隐藏文件、Obsidian 配置以及上述两个根目录之外的所有内容。
- `reports/*.pdf` 相对链接必须从同一公司研究中的财报索引解析为交易所或公司官网 URL；无法找到官方映射时发布失败，不生成断链。
- 拒绝包含本机绝对路径、Obsidian 双链/嵌入、YAML frontmatter、秘密凭据特征或越界相对路径的公开副本。
- 同步前后生成内容清单；被删除的源文件会从公开副本中同步删除，避免陈旧页面残留。

## 网站信息架构与渲染

首版采用 VitePress 作为确定性静态网站生成器。选择原因是它直接面向 Markdown、内置移动导航和本地搜索、支持 GitHub Pages 子路径，并允许用 Vue 主题组件实现 B 方案；不引入后端服务或运行时数据库。

站点结构：

- 首页：公司研究和产业专题两个入口，显示名称、证券代码或主题、数据核验日期和摘要。
- 公司专题页：以主报告为速览入口，顶部提供正文、财务、估值、竞对和其他材料导航；完整内容不删减。
- 行业专题页：按原目录组织文章，提供主题内目录和上一篇/下一篇导航。
- 全站搜索：使用构建时生成的本地索引，支持中文标题与正文搜索。
- 全局页脚：显示数据日期提示、来源责任和“不构成确定性投资建议”声明。

移动端规则：

- 正文字号、行高和内容宽度按手机阅读优先。
- 宽表保留全部列，放入明确可横向滑动的容器；首屏可显示关键列摘要，但不能删除原表信息。
- 页面目录在手机端折叠，专题导航保持可触达。
- 图片按容器缩放，并保留点击查看原图能力。
- 外部来源在新标签页打开；内部 Markdown 链接转换为站内路由。

## 自动化与权限

私有仓库工作流：

- 自动触发条件：`master` 分支收到 push，且公司研究、产业专题或私有仓库发布工作流发生变化。
- 同时提供 `workflow_dispatch` 手动重跑入口，用于首次全量发布、失败重试，以及公开仓库同步程序升级后的全量重建；正常内容更新不需要人工触发。
- 使用并发锁；新任务可以取消同一分支上尚未完成的旧任务。
- 检出私有内容和公开站点仓库，运行公开仓库版本化的同步与校验程序。
- 只有内容发生变化且全部校验通过时，才向公开仓库 `main` 提交。

跨仓库写入使用仓库级 Deploy Key：

- 公钥只配置在 `invest-research-site`，且仅授予该仓库写权限。
- 私钥只保存为 `obsidian-vault-invest` 的 Actions Secret。
- 不使用账号级 PAT，不赋予私有仓库读取权限，也不把密钥写入任何提交。

公开仓库工作流：

- `main` 更新后安装锁定依赖、执行内容校验、构建站点并上传 Pages artifact。
- 部署任务使用 GitHub Pages 所需的最小 `pages: write` 与 `id-token: write` 权限。
- 构建成功后才切换线上版本；失败时保留上一版本。

## 错误处理与验证

发布校验至少覆盖：

- 公开路径白名单和排除目录。
- Markdown/GFM 禁用语法扫描。
- 本地绝对路径、目录穿越与疑似秘密扫描。
- 内部 Markdown、图片、CSV 链接完整性。
- 本地 PDF 链接的官方 URL 映射完整性。
- 重复路由、空专题、缺少主公司报告等结构错误。

实施验证至少覆盖：

- 单元测试：路径筛选、PDF 链接改写、公司分组、路由生成和失败保护。
- 构建测试：全部现有 36 篇 Markdown、2 个公司专题、3 个产业专题成功构建。
- 桌面和手机浏览器测试：首页、搜索、中际旭创速览、长文目录、宽表、图片和跨文档链接。
- 自动化测试：模拟非目标目录 push 不触发发布；目标目录 push 触发同步；校验失败不产生公开提交。
- 线上验收：默认 GitHub Pages URL 使用 HTTPS，核心页面返回 200，刷新深层路由不产生 404。

## 首版不包含

- 登录、权限控制、评论、数据库和动态服务端接口。
- 实时行情抓取或在网页运行时调用大模型。
- 把完整 vault、关注池、个股决策、异动雷达或原始财报 PDF 公开。
- 新自定义域名配置。
